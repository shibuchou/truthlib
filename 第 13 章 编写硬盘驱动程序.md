### 13.1.2 创建磁盘分区表

本节的任务是把刚创建的磁盘 hd80M.img 分区

盘片：类似光盘中的一个圆盘，上面布满了磁性介质。

扇区：扇区是硬盘读写的基本单位，它在磁道上均匀分布，与磁头和磁道不同，扇区从 1 开始编号。扇区的

大小字节数=256×N，N 为自然数。通常取 N 为 2，因此扇区大小为 512 字节。“簇”或“块”这些是操作系统中读写数据的单位，并不是磁盘原生支持的，一个簇或块等于 1 个以上的扇区。因为磁盘本身就是整个机器的瓶颈，它是速度较低的设备，若操作系统总去访问这些低速设备就太浪费时间了，因此操作系统不可能一次只写一个扇区，为了优化 I/O，操作系统把数据积攒到“多个扇区”时再一次性写入磁盘，这里的“多个扇区”就是指操作系统的簇或块。

磁道：盘片上的一个个同心圈就是磁道，它是扇区的载体，每一个磁道由外向里从 0 开始编号。同一盘片上的每一个磁道上都由扇区组成，即磁道其实是一圈扇区。磁盘上的磁道数取决于制作工艺。离圆心近的磁道与最外圈的磁道周长肯定不一致，那这两种磁道上的扇区数老硬盘是一样的，新式硬盘中已经改进了，外圈磁道会容纳更多的扇区，在新硬盘中有个地址转换器来兼容老硬盘的扇区寻址，因此咱们依然可以认为硬盘中每个磁道上的扇区数一样多。

磁头：需要某个设备来读盘片上的数据，这个设备就是磁头。一个盘片分为上下两个面，各面都有一个磁头，因此一个盘片包括两个磁头，磁头号就表示盘面，平时所说的盘面号就是磁头号。虽然单个盘片的容量不断在增长，但其潜力毕竟是有限的。为了实现大容量，硬盘中必须由多个盘片来组成。既然有多个盘片，两个磁头就自然不够用了，肯定要有盘片×2 个磁头，磁头编号由上到下从 0 开始。

柱面：指多个磁头同时写入。也就是通常我们在写一个文件时，是由多个磁头同时写入到不同的盘面中编号位置相同的磁道上，采用并行的方式，读写速度是单盘的（磁头数）倍。这些由不同盘面上的编号相同的磁道（这些编号相同的同心圆大小一致）从上到下所组成的圆柱体的回转面就称为柱面，因此柱面的大小等于盘面数（磁头数）乘以每磁道扇区数。既然一组编号相同的磁道是柱面，而且柱面上的所有磁道号都相同，所以磁道号就称为柱面号。

分区：是由多个编号连续的柱面组成的，因此分区在物理上的表现是由某段范围内的所有柱面组成的通心环，并不是像“饼图”那种逻辑表示，当然若整个硬盘只有 1 个分区，那这个分区就是个所有柱面组成的圆柱体。分区不能跨柱面，也就是同一个柱面不能包含两个分区，一个柱面只属于一个分区，分区的起始和终止都落在完整的柱面上，并不会出现多个分区共享同一柱面的情况，这就是所谓的“分区粒度”。因此，分区大小等于“每柱面上的扇区数”乘以“柱面数”，这就是我们实际分区时，键入的大小往往与实际大小不同的原因，分区大小总会是“每柱面上的扇区数”的整数倍，也就是会以柱面向上取整。假如硬盘上有 n 个柱面，1～10 柱面是 a 分区，11～23 是 b 分区，这两个分区不共享 11 号柱面。

现在得出公式。

**（1）硬盘容量=单片容量×磁头数。**

**（2）单片容量=每磁道扇区数×磁道数×512 字节。**

磁道数又等于柱面数，因此将公式 2 代入公式 1 后：

**硬盘容量=每磁道扇区数×柱面数×512 字节×磁头数**

### 13.1.3 磁盘分区表浅析

磁盘分区表（Disk Partition Table）简称 DPT，是由多个分区元信息汇成的表，表中每一个表项都对应一个分区，主要记录各分区的起始扇区地址，大小界限等。磁盘分区表就是个数组，此数组长度固定为 4，数组元素是分区元信息的结构。

![image-20251106144032080](F:\OS\xuexi\truth\图片\image-20251106144032080.png)

**硬盘最前面的 512 B 长什么样**

| 字节偏移      | 大小  | 内容                          | 常见值/例子                   |
| :------------ | :---- | :---------------------------- | :---------------------------- |
| 0x000 ~ 0x1BD | 446 B | 引导代码（boot code）         | 全 0 也可，装 GRUB 时才写指令 |
| 0x1BE ~ 0x1FD | 64 B  | **分区表**（4 个表项 × 16 B） | 本节重点，下面逐字节解剖      |
| 0x1FE ~ 0x1FF | 2 B   | 魔数，必须是 55 AA            | 小端显示 AA 55                |

```
00001B0: 00 00 00 00 00 00 00 00  62 7F F6 FD 00 00 01 00
00001C0: 01 00 83 0F 3F 1F 3F 00  00 00 C1 7D 00 00 00 00
00001D0: 00 00 00 00 00 00 00 00  00 00 00 00 00 00 00 00
00001E0: 00 00 00 00 00 00 00 00  00 00 00 00 00 00 00 00
00001F0: 01 20 05 0F 3F A1 00 7E  00 00 E0 FF 01 00 55 AA
```

- 0x1BE 开始 16 B → 第 1 个主分区
- 0x1CE 全 0 → 第 2 个表项空
- 0x1DE 全 0 → 第 3 个表项空
- 0x1EE 开始 16 B → 第 4 个表项，类型 0x05 扩展分区
- 最后 2 B 0x55AA 正确

 **16 字节表项记录了什么**

| 偏移 | 长度 | 含义                       | 例子值（十六进制） | 备注                                |
| :--- | :--- | :------------------------- | :----------------- | :---------------------------------- |
| 0x00 | 1 B  | 活动标志                   | 0x80               | 0x80=可启动，0=不可                 |
| 0x01 | 1 B  | 起始磁头                   | 0x01               | CHS 地址，老 BIOS 用                |
| 0x02 | 2 B  | 起始扇区+柱面              | 0x00 0x0F          | 位场拆分，现在基本不用              |
| 0x04 | 1 B  | **分区类型**               | 0x83               | 83=Linux，05=Extended，0F=W95 Ext'd |
| 0x05 | 1 B  | 结束磁头                   | 0x0F               | 同上，老 BIOS                       |
| 0x06 | 2 B  | 结束扇区+柱面              | 0x3F 0x1F          | 同上                                |
| 0x08 | 4 B  | **起始 LBA（相对整个盘）** | 0x0000003F         | **重点，下面全用它算绝对地址**      |
| 0x0C | 4 B  | **本分区的扇区总数**       | 0x00007DC1         | 容量 = 总数 ×512 B                  |

**真正有用的就 8 个字节——“类型 0x04 | 起始 LBA 0x08 | 扇区数 0x0C”**，其余都是为了兼容 1981 年的BIOS。

 **4 个表项用完，扩展分区 + 链式 EBR**

把 4 个表项中任意一个的类型写成 0x05，它就变成“总扩展分区”。

这块“总扩展分区”内部再切成若干**子扩展分区**，每个子扩展分区**前 512 B 又是一个 EBR**，结构完全克隆 MBR：446 B 空代码 + 64 B 分区表 + 55 AA。

EBR 里的分区表只用两项：
① 第一项 → 描述**本段逻辑分区**（类型 0x83、0x0F …）
② 第二项 → 指向下一个 EBR 的 LBA（类型仍是 0x05）

第三、四项全 0。
于是形成**单向链表**，BIOS/操作系统只要沿着第二项一路追下去就能找到所有逻辑分区。

**地址基准——“相对”变“绝对”的口诀**

| 分区种类   | 相对基准                 | 换算公式                           |
| :--------- | :----------------------- | :--------------------------------- |
| 主分区     | 0 号扇区                 | 绝对 LBA = 表项.LBA                |
| 总扩展分区 | 0 号扇区                 | 同上                               |
| 子扩展分区 | 总扩展分区的绝对 LBA     | 绝对 LBA = ext_lba_base + 表项.LBA |
| 逻辑分区   | 所在子扩展分区的绝对 LBA | 绝对 LBA = 子扩展.LBA + 表项.LBA   |

ext_lba_base 是代码里一个全局变量，第一次进入 partition_scan 时把总扩展分区的起始 LBA 写进去，以后每次递归都拿它当“底座”。

| 分区        | 类型 | 表项内偏移 | 表项内扇区数 | 绝对起始 LBA（十六进制）          | 容量      |
| :---------- | :--- | :--------- | :----------- | :-------------------------------- | :-------- |
| 主分区 sdb1 | 0x83 | 0x0000003F | 0x00007DC1   | = 0x3F                            | ≈ 15.6 MB |
| 总扩展 sdb4 | 0x05 | 0x00007E00 | 0x0001FFE0   | = 0x7E00                          | 剩余全部  |
| 逻辑 sdb5   | 0x66 | 0x0000003F | 0x000046A1   | = 0x7E00+0x3F = **0x7E3F**        | ≈ 8.9 MB  |
| 逻辑 sdb6   | 0x66 | 0x0000003F | 0x00006231   | = 0x7E00+0x46E0+0x3F = **0xC51F** | ≈ 12.2 MB |
| …           | …    | …          | …            | …                                 | …         |

你可以用计算器把 LBA×512 得字节，再 ÷1024÷1024 得 MB，与 fdisk 输出完全一致。

 **为什么 EBR 不放在 0 扇区也能被找到？**

**查找入口永远是 0 扇区 MBR**。MBR 第 4 表项给出“总扩展分区”的起始 LBA（上表 0x7E00）。操作系统把这个 LBA 读进来，就是**第一个 EBR**。第一个 EBR 的第二表项又给出“下一个 EBR”LBA，如此链式推进。
**链表头固定，链表节点可任意多**，既突破 4 个分区限制，又不用改 MBR 格式 → 向上兼容 1981 年的 DOS。

**4 个 16 字节表项是户口本，0x05 是同意分家；后面每个 EBR 都是一页族谱，第二栏永远写着‘下一家在哪’**。

#### 为什么需要 thread_yield

1. 硬盘读/写命令发下去后，**31 s 内**盘才会回中断；
2. 当前线程不能空转占 CPU，需要**立即让出**去跑别的任务；
3. `thread_block()` 会把线程挂到阻塞队列，**必须由别人唤醒**；而 `thread_yield()` 只是把自己**重新放就绪队列尾**，**下次还能被调度**——**自发自收**，更适合“等硬盘”这种**短时间、可预期**的延迟。

#### 为什么再补一个 idle 线程

**旧问题**：就绪队列为空时 `schedule()` 会 ASSERT 宕机。
**新需求**：**没事干也别死机**，让 CPU 真正睡过去（hlt），**把功耗降到 0**。

#### 工作流程

1. 系统启动时 `thread_start("idle", 10, idle, NULL)` 创建 idle，**优先级最低**。
2. 正常任务都阻塞/挂起 → 就绪队列为空 → `schedule()` **主动唤醒 idle**。
3. idle 一被调度就执行 `sti;hlt`：**CPU 进入停机状态，中断来前 0% 占用**。
4. 任何中断（时钟/键盘/硬盘）一来 → CPU 醒 → 中断返回时若就绪队列不空，**调度器自然选更高优先级任务**，idle 再次被阻塞。

硬盘驱动可以**安心调用 `thread_yield`** 把 CPU 让出去，**不会空转**。系统空载时**CPU 利用率掉到 0%**，**功耗最低**。调度器**再无“空队列”ASSERT**，系统**7×24 小时待机也不死机**。

### 13.2.3 实现简单的休眠函数

#####  分割大传输

端口 8 位，Sector Count=0 才表示 256，因此**一次最多 256 扇区**。

while 循环把 `sec_cnt` 切成 256 的整数倍 + 余数，**效率与兼容性兼顾**。

##### 锁通道——防止“同一通道两盘同时忙”

```c
lock_acquire(&hd->my_channel->lock);   // 拿通道级互斥锁
... 选盘 → 发命令 → 阻塞 → 中断唤醒 → 放锁
lock_release(&hd->my_channel->lock);
```

**原因**：两盘共一根 IRQ，无法区分谁回中断，**干脆禁止并发**。

##### 阻塞点选择

| 读                                                           | 写                                           |
| :----------------------------------------------------------- | :------------------------------------------- |
| **命令已发出** → 盘开始内部寻道 → **再睡**                   | **数据写完端口** → 盘开始内部固化 → **再睡** |
| **策略**：让盘“真正忙”起来后再 `sema_down(&channel->disk_done)`，**把 CPU 让给别人**。 |                                              |

#####  错误处理`busy_wait` 返回 false → 直接 `PANIC(...)`，**驱动层不做重试**，简化逻辑。

#### 中断处理函数——intr_hd_handler

```c
void intr_hd_handler(uint8_t irq_no)
{
    uint8_t ch_no = irq_no - 0x2E;              // 0→ata0  1→ata1
    struct ide_channel *ch = &channels[ch_no];
    if (ch->expecting_intr) {                   // 的确是我在等的命令
        ch->expecting_intr = false;
        sema_up(&ch->disk_done);                // 唤醒阻塞在通道上的线程
        inb(reg_status(ch));                    // 读 Status 清中断
    }
}
```

**expecting_intr** 由 `cmd_out` 置位，**防止“ stray 中断”**误唤醒。

**必须读一次 Status 寄存器**，否则盘不会发下一次中断（ATA 规范）。

#### 在 ide_init 末尾注册中断向量

```c
register_handler(channel->irq_no, intr_hd_handler);
```

**注册完成 = 驱动闭环**：
发命令 → `sema_down` 阻塞 → 盘完事 → 中断 → `sema_up` → 唤醒继续读/写 → 放锁。

硬盘命名规则是[x]d[y][n]，其中只有字母 d 是固定的，其他带中括号的字

符都是多选值，下面从左到右介绍各个字符。

x 表示硬盘分类，硬盘有两大类，IDE 磁盘和 SCSI 磁盘。h 代表 IDE 磁盘，s 代表 SCSI 磁盘，故 x

取值为 h 和 s。

d 表示 disk，即磁盘。

y 表示设备号，以区分第几个设备，取值范围是小写字符，其中 a 是第 1 个硬盘，b 是第 2 个硬盘，

依次类推。

n 表示分区号，也就是一个硬盘上的第几个分区。分区以数字 1 开始，依次类推。

综上所述，sda 表示第 1 个 SCSI 硬盘，hdc 表示第 3 个 IDE 硬盘，sda1 表示第 1 个 SCSI 硬盘的第 1个分区，hdc3 表示第 3 个 IDE 硬盘的第 3 个分区。咱们这里统一用 SCSI 硬盘的命名规则来命名虚拟硬盘hd60M.img 和 hd80M.img。其中 hd60M.img 为 sda，hd80M.img 为 sdb。hd60M.img 是裸盘，没有文件系统和分区，因此咱们只处理 hd80M.img，将其上的主分区占据 sdb[1～4]，逻辑分区占据 sdb[5～]。

##### 递归扫描算法 `partition_scan(hd, ext_lba)`

**入口**：0 号扇区 = MBR

**循环 4 表项**：

`fs_type == 0x05` → 扩展分区

第一次（ext_lba==0）记下 `ext_lba_base = 表项.start_lba`

递归 `partition_scan(hd, ext_lba_base + 表项.start_lba)`

`fs_type !=0 && !=5` → 主分区或逻辑分区

`ext_lba==0` → 主分区 → 填 `hd->prim_parts[p_no++]`

`ext_lba!=0` → 逻辑分区 → 填 `hd->logic_parts[l_no++]`

**内存分配**：每次递归 `sys_malloc(sizeof(struct boot_sector))`，**防止栈溢出**（512 B×递归深度）。
