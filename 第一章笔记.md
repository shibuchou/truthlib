# 本书第一章主要内容
## 1.1 工欲善其事，必先利其器
#### 操作系统是什么？
操作系统是管理计算机硬件和软件资源的系统软件，通过集中化管理（如文件系统、内存管理、进程调度等）简化用户和应用程序的操作。

操作系统虽然是底层软件，但完全可以用高级语言（如 C）和工具链来开发。
不要畏惧底层开发，工具选对就能事半功倍。
## 1.2 我们需要哪些编译器
### 1.2.1 GCC（C语言编译器）
选择理由：开源、功能强大、跨平台、Linux 标准编译器。
支持语言：C、C++、Fortran、Java、Go 等。
### 1.2.2 NASM（汇编编译器）
选择理由：语法简洁、支持 Linux、输出格式多（如 ELF、bin）。
使用场景：操作系统启动引导、硬件交互、寄存器操作等必须用汇编的地方。

## 1.3 操作系统的宿主环境
### 1.3.1 什么是虚拟机
虚拟机的概念和作用：模拟硬件、隔离风险、便于调试。
推荐 Bochs 作为开发用虚拟机，支持调试和硬件模拟。（使用原因首先是开源，再者Bochs 是用软件逐条模拟 x86 指令的，因此它可以精确追踪每一条指令的执行过程，包括寄存器、内存、I/O 状态等。）其优点在于能够模拟跟主机不同的机种，例如在 Sparc 系统里模拟 x86，但缺点是它的速度却慢得多。

### 1.3.2 虚拟机中再装一个虚拟机
提出“盗梦空间式”的开发方案：
在 VirtualBox 中安装 CentOS 6.3（Linux系统）。
在 CentOS 中安装 Bochs，用于运行和调试自己写的操作系统。
而我是在Windows中安装vmware，再在vmware中安装ubunutu，再安装bochs

### 1.3.3 VirtualBox 下载与安装
提供下载地址和安装建议，适用于 Windows、macOS、Linux。
### 1.3.4 Linux 发行版下载
推荐使用 CentOS 6.3，并提供镜像下载地址（如网易镜像、vault.centos.org）。
我使用了vmware配置ubunutu版本22.04

### 1.3.5 Bochs 下载与安装
源码安装：推荐从源码编译安装，以便开启调试功能。
安装步骤：
下载源码包（如 bochs-2.6.2.tar.gz）。
使用 ./configure 配置（开启调试器、反汇编器、图形界面等）。
make && make install 编译并安装。
常见问题：
报错 undefined reference to 'pthread_create' → 需要链接 pthread 库，修改 Makefile 添加 -lpthread。
## 1.4 配置 Bochs
通过配置文件 bochsrc.disk 设置虚拟机的硬件参数：
内存大小（32MB）
BIOS 和 VGA BIOS 路径
启动方式（硬盘启动）
硬盘配置（hd60M.img）
日志输出
开启调试器或 GDB 支持
## 1.5 运行 Bochs
启动 Bochs 并加载配置文件。
初次运行会报错“没有启动盘”，于是使用 bximage 工具创建虚拟硬盘。
将生成的硬盘配置写入配置文件，再次启动即可进入模拟环境。

## 配置如下:（注意修改目录）
```
#第一步，首先设置 Bochs 在运行过程中能够使用的内存，本例为 32MB。  
#关键字为：megs  
megs: 32
#第二步，设置对应真实机器的 BIOS 和 VGA BIOS。  
#对应两个关键字为：romimage 和 vgaromimage  
romimage: file=/home/shibuchou/truth/bochs/share/bochs/BIOS-bochs-latest  
vgaromimage: file=/home/shibuchou/truth/bochs/share/bochs/VGABIOS-lgpl-latest  
#第三步，设置 Bochs 所使用的磁盘，软盘的关键字为 floppy。  
#若只有一个软盘，则使用 floppya 即可，若有多个，则为 floppya，floppyb…  
#floppya: 1_44=a.img, status=inserted
#第四步，选择启动盘符。  
#boot: floppy  
#默认从软盘启动，将其注释  
boot: disk
#改为从硬盘启动。我们的任何代码都将直接写在硬盘上，所以不会再有读写软盘的操作。  
#第五步，设置日志文件的输出。  
log: bochs.out
#第六步，开启或关闭某些功能。
#下面是关闭鼠标，并打开键盘。  
mouse: enabled=0  
keyboard: keymap=/home/shibuchou/truth/bochs/share/bochs/keymaps/x11-pc-us.map  
#硬盘设置  
ata0: enabled=1, ioaddr1=0x1f0, ioaddr2=0x3f0, irq=14  
ata0-master: type=disk, path="hd60M.img", mode=flat,cylinders=121,heads=16,spt=63  
#下面的是增加的 bochs 对 gdb 的支持，这样 gdb 便可以远程连接到此机器的 1234 端口调试了
#gdbstub: enabled=1, port=1234, text_base=0, data_base=0, bss_base=0
################### 配置文件结束 #####################
```

这里尤其注意目录在bochs的目录下，后面要跟share/bochs

## 创建60MB启动磁盘:

依次输入:  

>| 输入内容      | 含义                                                |
>| ------------- | --------------------------------------------------- |
>| `bin/bximage` | 启动 Bochs 的磁盘镜像创建工具                       |
>| `1`           | 选择创建 **硬盘（Hard Disk）**                      |
>| `hd`          | 指定镜像类型为 **硬盘**（不是软盘）                 |
>| `flat`        | 选择存储格式为 **flat（线性连续文件）**，适合启动盘 |
>| `60`          | 设置磁盘大小为 **60MB**                             |
>| `hd60M.img`   | 设置镜像文件名为 `hd60M.img`，最终保存在当前目录    |

## 进行代码测试
```
;主引导程序   
;------------------------------------------------------------    
 SECTION MBR vstart=0x7c00   
 mov ax,cs   
 mov ds,ax   
 mov es,ax   
 mov ss,ax   
 mov fs,ax    
 mov sp,0x7c00    
; 清屏利用 0x06 号功能,上卷全部行,则可清屏｡    
; -----------------------------------------------------------     
 ;INT 0x10 功能号:0x06 功能描述:上卷窗口  
;------------------------------------------------------     
 ;输入:   
 ;AH 功能号= 0x06   
 ;AL = 上卷的行数(如果为 0,表示全部)   
 ;BH = 上卷行属性  
 ;(CL,CH) = 窗口左上角的(X,Y)位置  
 ;(DL,DH) = 窗口右下角的(X,Y)位置  
 ;无返回值:   
 mov ax, 0x600  
 mov bx, 0x700  
 mov cx, 0 ; 左上角: (0, 0)  
 mov dx, 0x184f ; 右下角: (80,25),  
 ; VGA 文本模式中,一行只能容纳 80 个字符,共 25 行｡ 
 ; 下标从 0 开始,所以 0x18=24,0x4f=79   
 int 0x10 ; int 0x10   
;;;;;;;;; 下面这三行代码获取光标位置 ;;;;;;;;;   
 ;.get_cursor 获取当前光标位置,在光标位置处打印字符｡  
 mov ah, 3 ; 输入: 3 号子功能是获取光标位置,需要存入 ah 寄存器  
 mov bh, 0 ; bh 寄存器存储的是待获取光标的页号  
int 0x10 ; 输出: ch=光标开始行,cl=光标结束行  
 ; dh=光标所在行号,dl=光标所在列号  
;;;;;;;;; 获取光标位置结束 ;;;;;;;;;;;;;;;;   
;;;;;;;;; 打印字符串 ;;;;;;;;;;;   
 ;还是用 10h 中断,不过这次调用 13 号子功能打印字符串  
 mov ax, message   
 mov bp, ax ; es:bp 为串首地址,es 此时同 cs 一致,   
 ; 开头时已经为 sreg 初始化  
; 光标位置要用到 dx 寄存器中内容,cx 中的光标位置可忽略  
 mov cx, 5 ; cx 为串长度,不包括结束符 0 的字符个数  
 mov ax, 0x1301 ;子功能号 13 显示字符及属性,要存入 ah 寄存器,   
 ; al 设置写字符方式 ah=01: 显示字符串,光标跟随移动  
 mov bx, 0x2 ; bh 存储要显示的页号,此处是第 0 页,    
 ; bl 中是字符属性,属性黑底绿字(bl = 02h)   
 int 0x10 ; 执行 BIOS 0x10 号中断  
 ;;;;;;;;; 打字字符串结束 ;;;;;;;;;;;;;;;    
jmp $ ; 使程序悬停在此   
message db "1 MBR"   
 times 510-($-$$) db 0    
 db 0x55,0xaa  
```
