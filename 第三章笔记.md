###  3.1 地址、section、vstart （彻底搞懂代码在哪）

#### 3.1.1 什么是地址？  
1. **地址 = 符号在文件中的偏移量**  

​	编译器给每个变量、指令分配一个从文件头开始的“偏移量”  

​	反汇编工具（ndisasm）可直观看到：  
​     `	mov ax, [0x1234]` ← 这里的0x1234就是偏移量  

2. **数组下标从0开始** 的原因：第1个元素的偏移量是0  

3. 地址等于上一个地址+上一个地址处的内容的长度

   ![image-20250716214351828](F:\OS\xuexi\truth\图片\image-20250716214351828.png)

#### 3.1.2 什么是section？  
1. **逻辑分区**：把代码、数据、栈分开管理（增强可读性,NASM和CPU本身并不区分section，只是为了方便程序员阅读和维护）  

2. **不改变实际地址**：  
   无论是否用`section`，最终地址仍是**连续偏移量** ，其并没有对程序中的地址产生任何影响

```asm
section data
var1 dd 0x1234
section code
mov ax, [var1] ; 地址依旧是紧挨着前面
```

#### 3.1.3 什么是vstart？  

1. **“虚拟起始地址”**：

   告诉编译器：“下面这段代码/数据，将来会被加载到 **内存的某个固定地址**，请按这个地址重新计算偏移”  

2. **关键区别**：  

​	`vstart=0x7C00` → 编译器把所有内部地址都“加0x7C00”  

​	**不影响文件大小**（不会真的填充0）  

​	编译器只会规划代码，只会为程序编址，并不负责加载。

3. **与ORG的关系**：  
   `vstart`（NASM）  ≈  `org`（MASM）—— 只是“编址起点”的声明，**不是加载器**
   
3. vstart 的时机是：我预先知道我的程序将来被加载到某地址处。程序只有加载到非 0 地址时 vstart 才是有用的，程序默认起始地址是 0。

###  3.2 CPU的实模式

#### 3.2.1 CPU工作原理（宏观视角）  
**三大部件**：  

1. 控制单元（取指、译码、发信号）  
2. 运算单元（ALU，负责加减乘除、逻辑运算）  
3. 存储单元（寄存器 + L1/L2缓存）  

**执行循环**：  
`取指 → 译码 → 取操作数 → 执行 → 更新IP → 下一条`  

#### 3.2.2 实模式寄存器  
| 寄存器      | 位数 | 用途       | 示例                 |
| ----------- | ---- | ---------- | -------------------- |
| AX          | 16   | 累加器     | `mov ax, 0x1234`     |
| BX          | 16   | 基址寄存器 | `mov [bx], al`       |
| CX          | 16   | 循环计数   | `loop label`         |
| DX          | 16   | 数据/端口  | `out dx, al`         |
| SI/DI       | 16   | 字符串操作 | `movsb`              |
| SP          | 16   | 栈顶指针   | `push ax`            |
| BP          | 16   | 栈帧基址   | `[bp+4]` 访问参数    |
| CS/DS/ES/SS | 16   | 段寄存器   | `CS:IP` 形成20位地址 |

![image-20250717163315026](F:\OS\xuexi\truth\图片\image-20250717163315026.png)

在 16 位 CPU 中，只有一个附加段寄存器—ES。而 FS 和 GS 附加段寄存器是在 32 位CPU 中增加的。

![image-20250717163829412](F:\OS\xuexi\truth\图片\image-20250717163829412.png)

#### 3.2.3 内存分段由来

实模式的“实”体现在：程序中用到的地址都是真实的物理地址，“段基址：段内偏移”产生的逻辑地址就是物理地址，也就是程序员看到的完全是真实的内存。

**8086的困境**：  

- 地址总线20位 → 最多1MB  
- 寄存器16位 → 最多64KB  

**解决方案**：  
**段地址×16 + 偏移 = 20位物理地址**  
例：  
`0x1234:0x5678 → 0x12340 + 0x5678 = 0x179B8`  

**副作用**：  

段可重叠，同一物理地址可用多种段:偏移表示  

高端内存回卷（>1MB自动回0）

- **IP**：代码段内偏移  
- **FLAGS**：状态/控制位（ZF, CF, TF, IF …）

#### 3.2.4 20 位地址形成公式
```
物理地址 = (段寄存器 << 4) + 偏移
```
- **段寄存器值 × 16** 就是段基址。  
- 允许段重叠：同一段物理地址可用多组“段:偏移”表示。  

#### 实模式下的 8 种寻址方式
| 大类         | 细类               | 示例                  |
| ------------ | ------------------ | --------------------- |
| 寄存器寻址   | —                  | `mov ax, bx`          |
| 立即寻址     | —                  | `mov ax, 1234h`       |
| **内存寻址** | 直接               | `mov ax, [1234h]`     |
|              | 寄存器间接         | `mov ax, [bx]`        |
|              | 基址变址           | `mov ax, [bp+si]`     |
|              | 相对寻址           | `mov ax, [si+10h]`    |
|              | 基址+变址+相对寻址 | `mov ax, [bx+di+20h]` |

> **限制**：基址只能用 BX/BP；变址只能用 SI/DI；BP 默认段为 SS。

#### 3.2.5 栈机制
- **段寄存器 SS + 指针 SP** 定位栈顶。  
- **push/pop 指令** 自动修改 SP：  
  - **push ax**：`SP ← SP-2`，再存数据  
  - **pop ax**：先取数据，再 `SP ← SP+2`  
- **BP 寄存器**：把栈当普通数据段访问 `[bp±di sp]`，方便读局部变量/参数。

#### 3.2.6 & 3.2.7  过程调用与返回
| 指令          | 功能                    | 栈操作    |
| ------------- | ----------------------- | --------- |
| **call near** | 压入 IP → 跳转到目标    | 压 2 字节 |
| **call far**  | 压入 CS:IP → 跳转到目标 | 压 4 字节 |
| **ret**       | 弹出 IP → 返回          | 弹 2 字节 |
| **retf**      | 弹出 CS:IP → 远返回     | 弹 4 字节 |

> call/ret 必须配对；近远要一致，否则栈错位→崩溃。

#### 3.2.8 实模式下的 `jmp`（无条件跳转）

| 形式         | 汇编示例                | 机器码特征    | 说明              |
| ------------ | ----------------------- | ------------- | ----------------- |
| **短跳转**   | `jmp short label`       | `EB cb`       | 偏移 -128~+127    |
| **近跳转**   | `jmp near label`        | `E9 cw`       | 偏移 ±32K（段内） |
| **远跳转**   | `jmp far 0x1234:0x5678` | `EA ptr16:16` | 同时改 CS 和 IP   |
| **间接跳转** | `jmp word [bx]`         | `FF /4`       | 以内存字为偏移    |
| **远间接**   | `jmp dword [bx]`        | `FF /5`       | 内存双字=CS:IP    |

> 关键点：  
> - 只有 **far jmp** 能跨段；  
> - 在实模式下，**远跳转**会重新加载 CS，常用于跳转到不同段或加载内核。

#### 3.2.9 标志寄存器 `FLAGS`（16 位）

| 位   | 缩写 | 名称     | 含义（置 1 时）                   |
| ---- | ---- | -------- | --------------------------------- |
| 0    | CF   | 进位     | 无符号运算溢出                    |
| 2    | PF   | 奇偶     | 低 8 位 1 的个数为偶              |
| 4    | AF   | 辅助进位 | BCD 调整用                        |
| 6    | ZF   | 零       | 结果为 0                          |
| 7    | SF   | 符号     | 结果为负                          |
| 8    | TF   | 单步     | CPU 每执行一条指令就产生 1 号中断 |
| 9    | IF   | 中断允许 | 允许可屏蔽中断                    |
| 10   | DF   | 方向     | 字符串指令自减 (CLD/STD)          |
| 11   | OF   | 溢出     | 有符号运算溢出                    |

> 实战：  
> - `cli / sti` 可关/开 IF；  
> - 条件跳转 `je, jne, jg, jl` 等依赖 ZF/SF/OF/CF 组合。

#### 3.2.10 条件转移指令族

| 指令        | 条件          | 等价     | 典型用途     |
| ----------- | ------------- | -------- | ------------ |
| `je label`  | ZF=1          | `jz`     | 相等/为 0 跳 |
| `jne label` | ZF=0          | `jnz`    | 不等/非 0 跳 |
| `jg label`  | ZF=0 且 SF=OF | 有符号 > |              |
| `jl label`  | SF≠OF         | 有符号 < |              |
| `ja label`  | CF=0 且 ZF=0  | 无符号 > |              |
| `jb label`  | CF=1          | 无符号 < |              |

> - 带 **G/L** 的是**有符号**比较，带 **A/B** 的是**无符号**比较。  
> - 所有条件转移均为 **段内短跳转**（-128~+127）。

---

#### 3.2.11 实模式小结

1. 地址总线 20 位 → 物理空间 1 MB。  
2. **段×16+偏移** 是唯一寻址公式，所有寄存器都 16 位。  
3. **通用寄存器“约定俗成”**：AX 累加、CX 计数、DX 端口、BX/BP/SI/DI 寻址。  
4. **栈靠 SS:SP 定位**，push/pop 自动维护。  
5. **call/ret、jmp、条件转移** 提供流程控制。  
6. **FLAGS** 保存运算结果特征，条件跳转全部看它脸色。

###  3.3 直接操作显卡 （ BIOS中断实战  ）

#### 3.3.1 显卡文本模式  
80×25字符，每字符2字节（ASCII+属性）  

显存起始地址：`0xB8000`  

IO 接口的功能：

1．设置数据缓冲，解决 CPU 与外设的速度不匹配

2．设置信号电平转换电路

3．设置数据格式转换

4．设置时序控制电路来同步 CPU 和外部设备

5．提供地址译码

#### 3.3.2 BIOS中断 `int 0x10` 关键功能  
| AH   | 功能             | 输入参数                       | 示例            |
| ---- | ---------------- | ------------------------------ | --------------- |
| 06   | 上卷窗口（清屏） | AL=0, BH=属性, CX/DX=窗口坐标  | 清屏            |
| 03   | 获取光标         | BH=页号                        | 返回DH/DL=行/列 |
| 13   | 打印字符串       | ES:BP=字符串, CX=长度, AL=模式 | 显示"Hello"     |

####  3.3.3  显存、显卡、显示器关系  

**显存就是显卡上的 RAM**，在实模式下被**内存映射**到物理地址  

- 文本模式：`0xB8000` 开始  
- 图形模式：`0xA0000` 开始  

**显卡**负责把显存里的内容**实时转成视频信号**送到显示器。  

**CPU 不直接驱动显示器**——它只读写显存；显卡芯片负责扫描显存并生成图像。  

**文本模式下每个字符占 2 字节**：低字节=ASCII，高字节=属性（颜色、闪烁等）。

                                    ;BH = 上卷行属性
                                    ;(CL,CH) = 窗口左上角的(X,Y)位置

                                    ; 输出背景色绿色，前景色红色，并且跳动的字符串"1 MBR"
mov byte [gs:0x00],'1'
mov byte [gs:0x01],0xA4             ; A表示绿色背景闪烁，4表示前景色为红色

mov byte [gs:0x02],' '
mov byte [gs:0x03],0xA4

mov byte [gs:0x04],'M'
mov byte [gs:0x05],0xA4   

mov byte [gs:0x06],'B'
mov byte [gs:0x07],0xA4

mov byte [gs:0x08],'R'

jmp $		                        ; 通过死循环使程序悬停在此

times 510-($-$$) db 0
db 0x55,0xaa

   `int 10h, AH=06h` 上卷整个窗口（参数 AL=0 表示全部行）。  

2. **获取光标位置**：  

   `int 10h, AH=03h` 返回 DH/DL=行/列，避免字符覆盖。  

3. **打印字符串**：  

   `int 10h, AH=13h` 一次写多个字符；  

   输入：`ES:BP` 指向字符串，`CX`=长度，`BH`=页号，`BL`=属性（黑底绿字=0x02）。  

4. **死循环**：  

   `jmp $` 让 MBR 停在这里，防止 CPU 跑飞。

###  3.5 硬盘工作原理  

#### 3.5.1 硬盘结构  
**CHS寻址**：柱面(Cylinder) + 磁头(Head) + 扇区(Sector)  

**LBA寻址**：线性扇区号（0开始） 

####  3.5.2 硬盘工作原理

硬盘 = 若干片磁盘的“同心圆磁道” + 磁头 + 控制器，**数据按扇区 512 Byte 存取**，靠 **旋转+寻道+读写** 完成。  

1. 物理结构  
   
   盘片两面各有一磁头，多盘片→多磁头。  
   
   磁道 → 柱面（所有盘片同半径磁道的集合）。  
2. 存取单位  
   
   **最小单元：扇区（512 Byte）**；若干扇区＝簇/块。  
3. 工作步骤  
   1. 磁头移动到目标 **柱面**（寻道）。  
   2. 盘片旋转到目标 **扇区**（旋转延迟）。  
   3. 磁头 **读写 512 B** 数据流。  

> 因此，软件只需通过 CHS/LBA 指定“柱面-磁头-扇区”或“线性扇区号”，控制器会完成上述物理动作。

#### 3.5.4 常用的硬盘操作方法

1. 选择通道，往sector counter（该通道寄存器）写入待操作扇区数

2. 往该通道三个LBA寄存器写入扇区起始地址的低24位

3. 往device中会写入LBA地址的24~27位，并置低6位为1，使其为LBA模式，设置第4位，选择操作的硬盘（master或slave）

4. 往该通道上的command寄存器写入操作命令

5. 读取该通道上的status寄存器，判断硬盘工作是否完成

6. 以上若为读取硬盘则继续，否则结束

7. 将硬盘数据读出

   ##### 数据传输方法如下：

   （1）无条件传送方式。

   （2）查询传送方式。

   （3）中断传送方式。

   （4）直接存储器存取方式（DMA）。

   （5）I/O 处理机传送方式。

mov byte [gs:0x06],'B'
mov byte [gs:0x07],0xA4

mov byte [gs:0x08],'R'
mov byte [gs:0x09],0xA4

mov eax,LOADER_START_SECTOR	    ; 起始扇区lba地址
mov bx,LOADER_BASE_ADDR         ; 写入的地址
mov cx,1			            ; 待读入的扇区数
call rd_disk_m_16		        ; 以下读取程序的起始部分（一个扇区）

jmp LOADER_BASE_ADDR

                                ;-------------------------------------------------------------------------------
                                ;功能:读取硬盘n个扇区

rd_disk_m_16:	   
                                    ;-------------------------------------------------------------------------------
				                    ; eax=LBA扇区号
				                    ; ebx=将数据写入的内存地址
				                    ; ecx=读入的扇区数
    mov esi,eax	                    ;备份eax
    mov di,cx		                ;备份cx
                                    ;读写硬盘:
                                    ;第1步：选择特定通道的寄存器，设置要读取的扇区数
    mov dx,0x1f2
    mov al,cl
    out dx,al                       ;读取的扇区数

mov eax,esi	                    ;恢复ax

                                ;第2步：在特定通道寄存器中放入要读取扇区的地址，将LBA地址存入0x1f3 ~ 0x1f6
                                ;LBA地址7~0位写入端口0x1f3
mov dx,0x1f3                       
out dx,al                          

                                ;LBA地址15~8位写入端口0x1f4
mov cl,8
shr eax,cl
mov dx,0x1f4
out dx,al

                                ;LBA地址23~16位写入端口0x1f5
shr eax,cl
mov dx,0x1f5
out dx,al

shr eax,cl
and al,0x0f	                    ;lba第24~27位
or al,0xe0	                    ; 设置7～4位为1110,表示lba模式
mov dx,0x1f6
out dx,al

                                ;第3步：向0x1f7端口写入读命令，0x20 
mov dx,0x1f7
mov al,0x20                        
out dx,al

                                ;第4步：检测硬盘状态

.not_ready:
                                    ;同一端口，写时表示写入命令字，读时表示读入硬盘状态
    nop
    in al,dx
    and al,0x88	                    ;第4位为1表示硬盘控制器已准备好数据传输，第7位为1表示硬盘忙
    cmp al,0x08
    jnz .not_ready	                ;若未准备好，继续等。

                                ;第5步：从0x1f0端口读数据
mov ax, di                      ;di当中存储的是要读取的扇区数
mov dx, 256                     ;每个扇区512字节，一次读取两个字节，所以一个扇区就要读取256次，与扇区数相乘，就等得到总读取次数
mul dx                          ;8位乘法与16位乘法知识查看书p133,注意：16位乘法会改变dx的值！！！！
mov cx, ax	                    ; 得到了要读取的总次数，然后将这个数字放入cx中
mov dx, 0x1f0

.go_on_read:
    in ax,dx
    mov [bx],ax
    add bx,2		  
    loop .go_on_read
    ret

times 510-($-$$) db 0
db 0x55,0xaa
