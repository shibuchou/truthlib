# 第二章：编写MBR主引导记录



###  第2章核心任务

**从零开始，写出并运行第一个** **“主引导记录”（MBR）**，让CPU真正执行我们写的代码。

###  2.1 计算机的启动过程

**程序必须先“载入内存”才能执行**（CPU只能运行内存中的指令）。

**“载入”= 把程序复制到内存 + 把CPU的CS:IP指向程序入口**。

**启动流程接力**：  
`BIOS → MBR → 操作系统内核`

### 2.2 软件接力第一棒：BIOS

#### 2.2.1 实模式下的1MB内存布局

8086地址总线20位，最多访问1MB内存。

内存被严格划分区域（见表2-1）：

- 低地址：中断向量表、BIOS数据、可用内存

- 高地址：显存、ROM、BIOS代码

​	地址总线上会提前预留出来一些地址空间给这些外设用，这片连续的地址给显存，这片连续的地址给硬盘控制器等。留够了以后，地址总线上其余的可用地址再指向 DRAM，也就是指插在主板上的内存条、我们眼中的物理内存。这就是安装了 4GB 内存，电脑中只显示 3.8GB 左右的原因。

#### 2.2.2 BIOS如何苏醒

BIOS被写在ROM（块内存，被映射在低端 1MB 内存的顶部，即地址 0xF0000～0xFFFFF 处）中

**由硬件直接唤醒**：CPU加电后，CS:IP被强制设为 `0xF000:0xFFF0（等效地址将是 0xFFFF0）`。

**0xFFFF0处是一条跳转指令** `jmp far 0xF000:0xE05B`，真正BIOS入口。

![image-20250715210841260](F:\OS\xuexi\truth\图片\image-20250715210841260.png)

BIOS完成硬件自检、建立中断向量表，最后把接力棒交给MBR。

#### 2.2.3 为什么是0x7C00？

BIOS 最后一项工作校验启动盘中位于 0 盘 0 道 1 扇区的内容。

​	硬盘扇区的表示法有两种：CHS 方法，即柱面 Cylinder 磁头 Header 扇区 Sector，扇区的编号是从 1 开始的，不是 0，所以 0 盘 0 道 1 扇区其实就相当于 0 盘 0 道 0 扇区，它就是磁盘上最开始的那个扇区。而LBA 方式中，扇区编号是从 0 开始的。

​	如果此扇区的最后 2 个不是 0x55 和 0xaa，即使里面有可执行代码也无济于事了，BIOS 只执行扇区末尾的两个字节分别是魔数 0x55 和 0xaa代码（即主引导记录 MBR），加载到物理地址 0x7c00，随后跳转到此地址，继续执行。

**历史原因**：IBM PC 5150 BIOS把MBR加载到物理地址 `0x7C00`。

- **不能太低**：
  不能覆盖中断向量表（`0x00000–0x003FF`）和 BIOS 数据区（`0x00400–0x004FF`）。
- **不能太高**：
  不能和 DOS 内核（`IO.SYS`、`MSDOS.SYS`）冲突，这些文件加载在 **0x0070:0000**（即 `0x00700`）附近。
- **妥协结果**：
  IBM 工程师选了 **0x7C00**（≈ 32 KB 处），给 MBR 留出 **512 字节**，不会和 DOS 系统冲突。

**计算依据**：

DOS 1.0最小内存32KB（0x8000）。

减去MBR+栈空间1KB → 0x8000-0x400 = 0x7C00。

**结论**：0x7C00是一个“约定俗成”的地址，不是CPU规定。

###  2.3 让MBR先飞一会儿

#### 2.3.1 关键语法：$和$$

​	$和$$是编译器 NASM 预留的关键字，用来表示当前行和本 section 的地址，起到了标号的作用，它是 NASM 提供的，并不是 CPU 原生支持的。

`$` = 当前指令地址

`$$` = 当前section起始地址

用于计算填充空间、跳转地址等。

#### 2.3.2 NASM简单用法

```
nasm -f <format><filename> [-o <output>]
```

![image-20250715213930751](F:\OS\xuexi\truth\图片\image-20250715213930752.png)

输出格式：`nasm -f bin -o mbr.bin mbr.S`

推荐格式：

- `bin`：纯二进制（MBR用）
- `elf`：带文件头（后续内核用）

#### 2.3.3 MBR代码实战

#### 代码：
```
;主引导程序   
;------------------------------------------------------------    
SECTION MBR vstart=0x7c00   
mov ax,cs   
mov ds,ax   
mov es,ax   
mov ss,ax   
mov fs,ax    
mov sp,0x7c00    

; 清屏利用 0x06 号功能,上卷全部行,则可清屏｡    
; -----------------------------------------------------------     
;INT 0x10 功能号:0x06 功能描述:上卷窗口  
;------------------------------------------------------     
;输入:   
;AH 功能号= 0x06   
;AL = 上卷的行数(如果为 0,表示全部)   
;BH = 上卷行属性  
;(CL,CH) = 窗口左上角的(X,Y)位置  
;(DL,DH) = 窗口右下角的(X,Y)位置  
;无返回值:   
mov ax, 0x600  
mov bx, 0x700  
mov cx, 0 ; 左上角: (0, 0)  
mov dx, 0x184f ; 右下角: (80,25),  
                         ; VGA 文本模式中,一行只能容纳 80 个字符,共 25 行｡ 
                         ; 下标从 0 开始,所以 0x18=24,0x4f=79   
int 0x10 ; int 0x10   

;;;;;;;;; 下面这三行代码获取光标位置 ;;;;;;;;;   
;.get_cursor 获取当前光标位置,在光标位置处打印字符｡  
mov ah, 3 ; 输入: 3 号子功能是获取光标位置,需要存入 ah 寄存器  
mov bh, 0 ; bh 寄存器存储的是待获取光标的页号  

int 0x10 ; 输出: ch=光标开始行,cl=光标结束行  
; dh=光标所在行号,dl=光标所在列号  

;;;;;;;;; 获取光标位置结束 ;;;;;;;;;;;;;;;;   

;;;;;;;;; 打印字符串 ;;;;;;;;;;;   
;还是用 10h 中断,不过这次调用 13 号子功能打印字符串  
mov ax, message   
mov bp, ax ; es:bp 为串首地址,es 此时同 cs 一致,   
					 ; 开头时已经为 sreg 初始化  

                     ; 光标位置要用到 dx 寄存器中内容,cx 中的光标位置可忽略  
mov cx, 5 ; cx 为串长度,不包括结束符 0 的字符个数  
mov ax, 0x1301 ;子功能号 13 显示字符及属性,要存入 ah 寄存器,   
                              ; al 设置写字符方式 ah=01: 显示字符串,光标跟随移动  
mov bx, 0x2 ; bh 存储要显示的页号,此处是第 0 页,    
                        ; bl 中是字符属性,属性黑底绿字(bl = 02h)   
int 0x10 ; 执行 BIOS 0x10 号中断  
;;;;;;;;; 打字字符串结束 ;;;;;;;;;;;;;;;    

jmp $ ; 使程序悬停在此   

message db "1 MBR"   
times 510-($-$$) db 0    
db 0x55,0xaa 
```

**功能**：清屏 → 获取光标 → 打印字符串 "1 MBR"(背景色为黑色，前景色为绿色) → 死循环

**关键点**：

- 512字节，末尾必须是魔数 `0x55AA`（小端序 `0xAA55`）。
- 用BIOS中断 `int 0x10` 实现清屏和打印。
- 用 `times 510 - ($ - $$) db 0` 填充剩余空间。

### bios是什么

BIOS 全称叫 Base Input & Output System，即基本输入输出系统。  
在开机的一瞬间，也就是接电的一瞬间，CPU 的 cs：ip 寄存器被强制初始化为 0xF000：0xFFF0。此时BIOS启动  
BIOS的主要作用是检测、初始化硬件 硬并件提供了一些初始化的功能调用

### mbr是什么

MBR（主引导记录） 所在的位置是磁盘上最开始的那个扇区，0 盘 0 道 1 扇区  
大小必须为512byte多余的补0，最后两字节必须是0x55和0xaa

他是一个程序，存在于每一个磁盘中，但只有启动盘中的会被BIOS加载，

- 如果磁盘不是启动盘，MBR 里的 **引导代码（446 字节）通常是无意义的空代码或无效代码**。
- 只有 **被 BIOS 选为启动盘** 的磁盘，其 MBR 的引导代码才会被 **真正执行**。

MBR的引导代码接管后续操作，加载操作系统（如Windows/Linux），由于只有 446 字节，功能非常有限，通常只是“接力棒”——找到并加载更大的引导程序（比如 GRUB 的第二阶段或 Windows 的 Bootmgr）

| 项目           | 大小                                       |
| -------------- | ------------------------------------------ |
| **MBR 总大小** | **512 字节**（= 0.5 KB）                   |
| **引导代码区** | **前 446 字节**                            |
| **分区表**     | 接下来的 **64 字节**（4 个分区 × 16 字节） |
| **结束标志**   | 最后 **2 字节**（固定为 `55 AA`）          |

### 思考mbr与bios之间的关系是什么？为什么需要mbr？mbr在整个操作系统启动阶段中的作用是什么？

mbr是BIOS自检时需要调用的第一个程序，BIOS将MBR中的引导程序加载到内存并执行，从而将控制权交给MBR。  
MBR中包含分区表，记录了硬盘的分区信息（如分区的起始位置、大小等），操作系统通过这些信息识别硬盘的分区结构。  
在BIOS完成硬件初始化后，MBR中的引导程序被加载到内存并执行，MBR的引导程序会检查分区表的有效性，确保分区信息正确。

第二章主要让我们写出第一个裸机程序——MBR（主引导记录），讲解 BIOS 加电自检后把启动盘首扇区 512 字节读到内存 0x7C00 并跳转执行，我们据此用 NASM 写汇编清屏、打印“1 MBR”，再填充至 512 字节并以 0x55AA 结尾；随后用 dd 写入虚拟硬盘，在 Bochs 中运行，成功让 CPU 脱离 BIOS 运行我们的代码，完成操作系统启动第一步。
